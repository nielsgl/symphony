# Dashboard Client Packaging

## Source Of Truth

The dashboard browser client is authored under `src/api/dashboard-client/`.
`entry.ts` is the browser entrypoint and calls `startDashboardClient()` from
`app.ts`. Domain-facing modules such as `formatting.ts`, `connection.ts`,
`issues.ts`, `operator-actions.ts`, `project-history.ts`, and `stopped-runs.ts`
re-export importable functions for direct tests.

The served asset is generated by:

```bash
npm run build:dashboard-client
```

That script runs `scripts/build-dashboard-client.js`, which uses `esbuild` to
bundle `src/api/dashboard-client/entry.ts` into
`src/api/dashboard-assets/generated-client.ts`. The generated file is committed
so the runtime `renderDashboardClientJs(config)` API can return a string without
requiring a browser build step at request time.

## Runtime Wrapper

`src/api/dashboard-assets/client.ts` remains the public asset facade. It
normalizes `DashboardClientConfig`, writes the small runtime config globals used
by the browser bundle, and appends `GENERATED_DASHBOARD_CLIENT_JS`.

Those config globals are the only behavior-bearing wrapper strings left in the
dashboard client path. HTML and CSS assets still use template strings because
they are separate static assets and are not browser-client behavior.

- `src/api/dashboard-client/*.ts`: 3,328 authored lines
- `scripts/build-dashboard-client.js`: 33 lines
